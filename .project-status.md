## Important Decisions

- Architecture remains a modular monolith using Next.js App Router, TypeScript, Prisma, and MySQL.
- Business logic stays in services; route handlers remain thin.
- Actual files are stored on disk under `STORAGE_ROOT`; database stores metadata only.
- File upload uses browser multipart upload, stages files under `STORAGE_ROOT/temp/uploads`, then finalizes through the existing file service.
- Every uploaded file creates a `ProjectFile` and an initial immutable `FileVersion`.
- SHA256 checksum is used for duplicate detection.
- Project codes follow `PRJ-YYYY-NNN`.
- Project `projectCode` is immutable after creation.
- Soft delete remains preferred for historical data.
- Frontend uses a dark industrial dashboard style with restrained styling.
- No Prisma schema changes were needed after the initial database foundation.
